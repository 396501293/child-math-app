# 数学夜航 · 九九星图（乘法表记忆模式）规范（2026-07-04）

> 作者：Edu-PM ｜ 依据：题库难度与模式设计.md §二、四、五；技术设计规范 v1.1/v1.2；实际代码（types.ts `Progress`/`array-grid`、storage.ts 合并逻辑、generator 乘法干扰项、progression.ts `effectiveLevel`）。
> 面向：工程直接实现。规则权威落回题库文档体例；本模式无「档位」概念，故第 1、2 项替换为「会话结构+选题」「掌握度状态机」。

## 定位（关键决策）

- **它是第三个「模式」**，与无尽/限时并列进地图右侧入口（下方新增第三个按钮「九九星图」），**不是**第四章的关卡。理由：它有**跨会话持久的 per-fact 掌握度**（endless/timed 都是无状态练习场，它不是），架构上是 `Progress` 的又一个切片，天然是模式而非关卡。
- **独立进度，v1 不联动**：掌握度**不**回流主线星星/解锁，**不**影响 endless/timed 的 `effectiveLevel`。唯一的单向耦合是「哪些 fact 可练」由主线已学口诀表决定（ZPD：没学过 ×8 就不考 8×9），这是 主线→模式 的门控，不违反「掌握度不影响主线」。
- **口诀=fact 的单位**：`2×3` 与 `3×2` 算**同一 fact**「二三得六」（口诀单向、小在前）。理由：中文是靠背口诀记乘法，一条口诀即掌握两向；分开追踪等于凭空翻倍认知负荷、且与孩子真实记忆路径矛盾。**掌握度按口诀存**（canonical key `min×max`）；但**出题时两向都可出**（`2×3=?` 或 `3×2=?` 都练同一口诀），保证题面多样。
- **核心 fact = 36 条**：`2≤a≤b≤9`（C(8,2)+8=36，即传统「需要背的 36 句口诀」）。×1 行列平凡，**预点亮、不入题池**。「集齐星图」= 36 条全部点亮。

## 1. 会话结构与选题算法

- **一次会话 = 12 题**（约 3–4 分钟；也可中途点返回提前结算）。刻意短、可多次累积——这正是「慢慢记住」的载体。
- **活跃 fact 池** `active(p)`：核心 36 条中，`a` 或 `b` 属于「已学口诀表」`L(p)` 的那些。`L(p)` 由主线已得星（star≥1）的乘法档决定：档46→{2,5}、档47→+{3,4}、档49→+{6,7}、档50→+{8,9}、档51→{2..9}。（学会 ×2 表 = 会 2×2..2×9，故「含 2 的口诀」全部激活。）
- **每题按权重抽样**（权重按状态 `s`，见 §2）：
  - `s=0`（新/未见）: 权重 5，**但每会话最多引入 4 条新 fact**（「慢慢」的字面落实：新知识不一次倾倒）。
  - `s=1`: 权重 4（弱项优先）。
  - `s=2` 到期(`cd≤0`): 2.5 / 未到期: 0.8。
  - `s=3`（已点亮）到期: 1 / 未到期: 0.3（集齐后偶尔复习保温，轻量间隔重复）。
- **「再见面」而非扣分**：本题首次答错的 fact，**强制**在 `本题序号 + rand(3,5)` 处重新出一次（会话末尾则追加 1 题兜底），确保它以「这次答对了」收尾。
- **去重**：不与最近 3 题同 fact（「再见面」重排除外，那是刻意重逢）。
- 无 hardMode 语义（本模式不做缺数转换）；不进 endless/timed 题池。

## 2. 掌握度状态机

- **每 fact 存** `{ s: 0..3, cd: number }`（`s`=掌握度，`cd`=间隔冷却，单位「会话」）。未触碰的 fact 不落盘（缺省 = `s0`）。
- **升降规则**（每题作答时）：
  - **首次即答对**：`s = min(s+1, 3)`；`cd = s`（越熟休越久：s3 歇 3 个会话）。
  - **答错后本轮再见面答对**：`s` 不变（不因重试给满奖励）；`cd = 1`。
  - **答错（首选错项）**：`s = max(s-1, 0)`，微回落用于强化记忆，**孩子侧无任何可见惩罚**；触发 §1「再见面」。
- **会话结束**：所有已落盘 fact `cd = max(cd-1, 0)`（推进间隔重复时钟）。
- **grid 点亮映射**：`s0`=暗、`s1`=青绿微光（学习中）、`s2`=青绿亮（熟）、`s3`=琥珀金 `#F2A541`+pop（**点亮/掌握**）。「集齐星图」= 全 36 条 `s=3`。

## 3. 答案域与选项规则

- 题面固定为**乘积回忆** `a×b=?`（`a,b∈[2,9]`，答案域 `[4,81]`）。**可选深化**：对 `s≥2` 的 fact 有 30% 出**翻面缺数** `a×?=c`（复用 `missing-mul-b`），区分「真会背」与「顺口溜蒙」。
- 3 个选项 = 正确 + 2 干扰，去重乱序，**clamp [1,100]**，不出 0（沿用全局）。

## 4. 干扰项规则

- 直接复用第四章乘法干扰：**口诀邻位** `(a±1)×b`、`a×(b±1)`（背串行是口诀期最典型错误），不足回落距离干扰 `correct±d`，仍 clamp [1,100]、去重。generator 已实现，无新增。

## 5. 教具呈现（C/P 层）

- **阵列网格 `array-grid`（a 行 × b 列）常显**（不受 `showBlocks` 影响）——它是本模式的记忆锚，把「三四十二」锚到「3 排、每排 4 个 = 12」的图像。已在 types/generator 存在，复用。
- **星图本身即 P 层表征 + 内在激励**：9×9 表格盘是进度地图，点任意格 → 弹小卡：算式 `3×4=12` + 传统口诀「三四十二」+ mini 阵列点阵 + 🔊。已学未点亮可点、未学格更暗带锁但也可点预览（听口诀、看阵列），把「集齐表格」做成收集欲。
- **答对反馈**：揭示口诀 + 对应 grid 格「飞入点亮」动画（集贝入图），是胜任感出口。

## 6. 语音文案模板

- **决策：口诀句式为主、算式并排**。口诀是记忆编码、算式是 A 层语义，配对即 A 层桥；但口诀含答案，不能用作提问，故**分工**：
  - **提问**（算式式，不泄答案）：`「{a} 乘 {b} 等于几？」`；缺数变体 `「{a} 乘几等于 {c}？」`。
  - **揭示/听口诀/答对**（传统口诀式）：`product<10 → 「{a子}{b子}得{积}」`，`≥10 → 「{a子}{b子}{积}」`，**小在前、用汉字数**（如 2×3→「二三得六」、3×4→「三四十二」、7×8→「七八五十六」）。阵列提示 `「{a} 排，每排 {b} 个，一共几个？」`。
  - **集齐/点亮**：点亮一格「这句你记住啦！」；集齐 36 句「星图点亮！九九口诀你全会了！」（庆祝，非里程碑压力）。

## 7. 入口与解锁位置

- **解锁**：完成第 46 关（首次遇 ×，L={2,5}）后地图右侧点亮第三入口「九九星图」，角标「新玩法！」。早解锁使记忆与学习同步生长；星图随主线推进逐行「解锁 → 可点亮」。
- **不进** endless/timed 题池；**不**改 hardMode 语义；**不**动主线星星/解锁。
- **持久化**：`Progress` 新增切片 `timesTable`（endless/timed 的先例）——
  ```ts
  timesTable: {
    facts: Record<string, { s: 0|1|2|3; cd: number }>; // key = `${min}×${max}`，稀疏存
    sessions: number;      // 已完成会话数（间隔重复时钟）
    litBest?: number;      // 历史最多点亮数（结算展示，可选）
  }
  ```
  `defaultProgress()` 补 `{ facts:{}, sessions:0 }`；`loadProgress` 合并处比照 `endless/timed` 加一行 `timesTable: {...d.timesTable, ...parsed.timesTable}`。**不升 version**（v2 合并-over-defaults 天然向后兼容），无新 localStorage key。

## 8. 验收标准（孩子行为层面可观察）

- 孩子**主动跨天多次**开这个模式（≥2 天各 1 会话），而非一次刷完——「慢慢累积」成立。
- 星图**逐渐点亮**：一周内已学口诀表对应格从暗 → 金，孩子会指着金格说「这句我会了」。
- 答错时**无退缩/沮丧**：看完口诀+阵列跟读，稍后「再见面」能答对并露出「这次对了」的反应。
- 会主动**点格听口诀/看阵列**（把它当探索而非考试）。
- 反向验证：翻面缺数 `3×?=12` 也答得出（真记住，非按 `a×b` 顺序蒙）。

## UI 要点（复用 token 与屏幕结构）

- **屏幕结构复用**：同 `#stage` 1024×768、同背景渐变、`scale()` 缩放。**星图屏**新增（模式主页）；**答题屏复用 Quiz** 结构（题目行=算式、`array-grid` 教具、3 选项），仅顶栏换成「本轮 n/12 + 已点亮 X/36」，无连对/无计时。
- **9×9 表格盘尺寸估算（1024×768 可行）**：格子复用计数块 token **54×54 圆角 12**，gap 6；含 1 行 + 1 列表头共 **10×10** → `10×54 + 9×6 = 594px`（宽=高）。顶栏 top:32 + 高 ~60 = 92；盘顶留白后 `92 + 594 + 底 40 ≈ 726 < 768` ✓。盘置左（x:40，占 594 宽，止于 634），右侧复用地图右面板宽 **280**（吉祥物一句话 + 「开始一轮」CTA + 听口诀预览卡）：`40+594+30+280 = 944 < 1024`（右留 80）✓。
- **格状态色**：暗 `rgba(255,255,255,.06)`（未点亮）→ 青绿系 `#2FB29B`（学习中/熟）→ 琥珀金 `#F2A541`+pop（点亮）；未学表格更暗 + 小锁角标，仍可点预览。表头 1–9 用琥珀字。
- **点格弹卡**：`rgba(255,255,255,.07)` 圆角 20，内含算式 + 传统口诀（大字）+ mini 阵列点阵 + 🔊；沿用 pop .35s。
- **CTA/按钮**：沿用红 `#E85D5D` 圆角 22 硬阴影范式；模式入口按钮沿用 `rgba(255,255,255,.1)` 底范式（同 endless/timed）。
- **答对动画**：新增「格飞入点亮」——被答对 fact 的格从题目行位置飞向星图对应坐标并点亮（内在激励「集齐」的视觉兑现）。

## 附：实现歧义裁决验收（2026-07-04，commit 21f3b16）

1. **掌握度每 fact 每会话最多变动一次** —— 通过，且优于字面规范。跨会话累积才是分布式重复；一次会话 s0→s3 是「一次刷熟」的假象，与 cd 的会话单位及「一周暗→金」验收矛盾。净每会话约升一级为本模式正解。
2. **onWrong 置 cd=0** —— 通过。cd 是熟练项的「休息」，不是错题的锁定期；刚错的口诀必须立即可练，与弱项优先一致。
3. **缺数变体不带阵列** —— 通过。全网格可数出被隐藏因子，检验失效；测记忆意图（§3/§8）优先于教具常显（§5）。补充一条：答错后的**揭示阶段**仍应展示口诀+阵列（检验在提问、支撑在揭示，两者不冲突）。
4. **提问用阿拉伯数字、口诀留给揭示** —— 通过。提问本就是算式式（§6 分工），数字书写形态属呈现层，TTS 朗读一致；口诀句式的记忆编码职能不受影响。
5. **再见面每 fact 限一次、不循环** —— 通过。无限重排会滑向「强制重刷」的惩罚感；错后 s 已降、cd=0，下会话高权重自然再来，间隔重来恰好比当场磨更利于记忆。
