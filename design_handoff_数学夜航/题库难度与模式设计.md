# 数学夜航 · 题库、难度与玩法模式设计

> 本文档细化并**替代** `README.md` 中「出题规则」一条，并扩展为 **3 章 × 15 关 = 45 关**主线 + 两个玩法模式（无尽 / 限时）。
> 目标年龄从「5 岁」扩展为 **4–7 岁（学前中大班到一年级）**：第一章保持 5 岁无压力设计，第二、三章难度上探。
> 视觉与交互仍以 `数学夜航原型.dc.html` 为准（README 的「章节内蛇形 15 关」结构天然支持多章，每章地图布局不变）；新增 UI 见文末清单。

## 设计原则

1. **无惩罚**：任何模式下答错都不扣分、不失败（限时的「时间到」是自然收尾），仅原地重试。
2. **数据驱动**：全局 45 个难度档（band），每档一条配置；主线、无尽、限时共用同一约束生成器，加章节只加配置。
3. **主线是学习主脉，模式是练习场**：新题型只在主线引入；模式只练已解锁的内容，自动适龄。
4. **每章 = 5 段 × 3 关**：段内逐关递增，相邻段约束重叠过渡，不出现难度断层。

## 二、玩法模式

入口在地图右侧面板（见「新增 UI 清单」）。

### 2.1 无尽模式「无尽夜航」

| 项 | 规则 |
|---|---|
| 解锁 | 完成第 3 关后入口点亮 |
| 出题 | 连续出题不分关。**起始档 = 当前章的第 1 档**（进入第二章后从档 16 起步，避免高年龄段冗长热身）；**每答对 4 题升一档**，升至「主线已解锁的最高档」后停留；不出还没见过的题型 |
| 答错 | 与主线相同：原地重试、选项排除；额外效果仅为**中断当前连对** |
| 结束 | 无失败。孩子点返回键结束本轮 → 无尽结算屏 |
| 计分 | 本轮答对数（主题化为「航行距离」，每答对一题小船前进一格）+ 当前连对 🔥 |
| 纪录 | 持久化最高连对 `bestStreak`、历史累计答题数 `totalAnswered`；破纪录时结算屏特别庆祝 |
| 去重 | 避免与最近 5 题重复 |

### 2.2 限时模式「星光冲刺」

| 项 | 规则 |
|---|---|
| 解锁 | 完成第 9 关后入口点亮 |
| 计时 | 起始 60s；**答对 +8s，上限 90s**；答错不扣时 |
| 出题 | 从**当前章及前一章中已完成的档**均匀混抽（练熟练度，不引入新知识，难度不爬坡；跨度限两章，避免一年级孩子还在抽 2+3）；避免与最近 5 题重复 |
| 结束 | 时间到 → 庆祝式结算「时间到！你答对了 N 题！」——无任何失败措辞 |
| 纪录 | 持久化单轮最多答对数 `bestCount`；破纪录庆祝 |
| 教具 | 默认**隐藏计数块**（鼓励心算）；可在 Tweaks 打开 |
| 温和提示 | 剩余 10s 时时间条变珊瑚红 `#E85D5D`；**不加急促音效** |

### 2.3 模式流程

```
地图 ─▶ 无尽夜航：出题→答题 ×∞（档位随连对爬升）─ 点返回 ─▶ 无尽结算 ─▶ 地图
地图 ─▶ 星光冲刺：60s 倒计时，出题→答题（答对+8s）─ 时间到 ─▶ 冲刺结算 ─▶ 地图
```

## 三、约束生成器（主线与模式共用）

每档配置形如：

```js
// 档 17（退位减法）
{ band: 17, type: 'sub', aRange: [11, 18], bRange: [1, 9], borrow: true }
// 档 27（加减混连）
{ band: 27, type: 'chain3', ops: ['+-', '-+'], stepRange: [1, 20] }
// 档 44（两位数进退位）
{ band: 44, mix: [ { type: 'add', twoDigit: true, carry: true,  sumMax: 99, weight: 0.5 },
                   { type: 'sub', twoDigit: true, borrow: true, resMin: 1,  weight: 0.5 } ] }
```

全局约束：

- 操作数 ≥ 1（不出 `+0`、`−0`）；减法与连算的**每一步中间结果 ≥ 1**；缺数题 `?` ≥ 1。
- 答案范围：第一、二章 [1, 20]；第三章 [1, 100]（仅档 31/33 的整十加法可达 100）。
- 主线：整关题面去重（操作数与运算符组合两两不同）；生成后按答案从小到大排序（第一题永远不是本关最难）；重玩重新随机生成。
- 模式：滚动去重（不与最近 5 题重复）。
- 解锁规则线性贯穿三章：完成第 n 关解锁 n+1（上限 45），完成第 15/30 关即进入下一章地图。

## 四、干扰项规则（替代原「±2 取 3 个」）

3 个选项 = 正确答案 + 2 个干扰项，去重后乱序。

候选池（按优先级）：

1. **距离干扰**：`correct ± d`。档 1–6 取 d ∈ {2, 3}（初学阶段拉开区分度）；档 7 起取 d ∈ {1, 2}（要求精确点数）。
2. **弄反干扰**（档 7 起，50% 概率替换一个距离干扰项）：加减弄反的结果，如 `8−3` 的题给 11。
3. **十位偏差干扰**（第三章专属，优先级同弄反）：`correct ± 10`——针对 100 以内最常见的「十位算错」，如 `34+25` 的题给 49 或 69。
4. **兜底**：候选不足时逐步扩大 d 至 3、4……

所有干扰项 clamp 到 **[1, 章数值上限]**（第一、二章 20，第三章 100）；选项不出现 0，与「不出 0」原则一致。

边界用例（实现后应逐条验证）：

| 情形 | 结果 |
|---|---|
| correct = 1（如 3−2，档 3，d∈{2,3}） | 负侧全被 clamp 掉，正侧 {3, 4} 恰好 2 个 ✓ |
| correct = 19（档 10/15，d∈{1,2}） | {17, 18, 20}，21 被 clamp 掉 ✓ |
| correct = 100（档 31，整十题） | {90, 98, 99}（110 被 clamp 掉；整十题建议干扰项偏好 ±10 → 90、80）✓ |
| 弄反 / 十位偏差结果越界或撞正确答案 | 丢弃，回落距离干扰 ✓ |

**整十题特例**：距离干扰改用 d ∈ {10, 20}（选项 30/40/60 比 39/40/41 更符合该题型的心智动作）。判定按**题面**（第三章且全部操作数为整十）而非档位——主要出现在档 31–33，第三章其他档偶发的全整十题面同样适用。

## 五、计数块教具适配

| 题型 | 教具行为 |
|---|---|
| 第一、二章加/减/缺数 `a+?=c` | 沿用原型：加法两色分组、减法点击划掉、缺数空槽填充 |
| 缺数 `?+b=c` | 与 `a+?=c` 同构：c 个槽位中 b 个实心（琥珀），待填部分在前（青绿空槽） |
| 缺数 `a−?=c` | 显示 a 个实心块，其中 c 个带「保留」标记；孩子划掉若干块，划掉数即答案 |
| 连加/混连（档 25–27） | 三组分色：青绿 / 琥珀 / 白 `rgba(255,255,255,.85)`（第三色为新增设计项） |
| 第三章全部 | **默认隐藏**（100 以内逐块点数无教学价值）；后续设计项：「十条 + 散块」教具（1 条 = 10，最多 9 条 + 9 散块） |
